Это окно позволяет получить быстрый доступ к последним открытым документам, создавать новые документы различных форматов и создавать документы на основе шаблонов.

h1. Создание новых документов

Для создания нового документа нажмите сочетание горячих клавиш [Ctrl] + [N] или выберите пункт главного меню _File > New…_

В открывшемся диалоговом окне New Document необходимо выбрать категорию создаваемого документа, тип документа и, при необходимости,  дополнительные параметры. Для документов в формате HTML можно также указать параметр DocType и присоединить одну или несколько таблиц стилей.

Параметры создания новых документов задаются в настройках программы (в главном меню _Edit > Preferences > New Document_ или сочетание горячих клавиш [Ctrl] + [U]  и выбрать категорию _New Document_).

h4. Рис. 2. Окно создания нового документа

Новый html-файл будет открыт в DreamWeaver в последнем использовавшемся режиме отображения документов.

Если были неверно заданы параметры версии стандарта документа (Doctype) или кодировка, их можно исправить, выполнив команду главного меню _Modify> Page Properties…_ или нажав сочетание горячих клавиш [Ctrl] + [J].  Указанные свойства меняются в категории _Title/Encoding_.

h1. Интерфейс программы _Adobe DreamWeaver_ CS5

Интерфейс программы _Adobe DreamWeaver_ включает в себя окна документов, панели инструментов, палитры и инспекторы.

В рабочем пространстве программы имеются три закрепленных области (дока), в которых располагаются группы палитр и инспекторы. Эти области расположены слева, справа и снизу от окна документа. Нажатием на кнопку скрытия группы палитр можно быстро изменить площадь отображения рабочего документа. Чтобы полностью скрыть правый и нижний доки палитр надо нажать [F4]. Повторное нажатие [F4] включит отображение блоков палитр.

* объединять в группы;

h3. Плавающие и пристыкованные палитры

По умолчанию палитры в DreamWeaver располагаются группами в закрепленных областях (доках). В случае необходимости палитру или группу палитр можно «отстыковать» и сделать свободно перемещаемой по экрану. Для этого необходимо поднести курсор к ярлычку палитры и перетащить ее на свободное место.

Для закрепления плавающей палитры в доке палитр нужно поднести курсор к ярлычку палитры и перетащить ее в блок, пока ее контур не покажет, что она закреплена.

h3. Группы палитр

По умолчанию некоторые палитры объединены в группы по принципу общности задач (например _Files_ / _Assets_ / _Snippets_).

* для вынесения палитры из группы — потащите за ярлычок палитры на разделительную линию между палитрами в блоке.
* для объединения палитры в группу с другой палитрой — потащите за ярлычок палитры к другой палитре.

h3. Сворачивание и максимизация палитр

Для экономии пространства рабочей области редко используемые группы палитр можно свернуть. Для отображения максимально возможного количества информации палитру можно максимизировать (часто требуется для палитр Files, CSS Styles).

* для сворачивания палитры/группы палитр – дважды щелкните по ярлычку палитры.
* для максимизации палитры – один раз щелкните по ярлычку палитры.

h2. Панели инструментов

В программе _Adobe DreamWeaver_ имеется пять панелей инструментов (Toolbars):

* Style Rendering;
* Document;
* Standart;
* Browser Navigation;
* Coding.

Панели инструментов *Standart* и *Document* предназначены для выполнения стандартных операций редактирования и операций с документами, панель инструментов *Coding* — для быстрого ввода кода, а панель *Style Rendering* — для просмотра страницы в том виде, в котором она будет отображена на других типах носителей. Панель инструментов *Browser Navigation* становится активной при интерактивном просмотре, и в ней отображается адрес страницы, находящейся в окне документа.

При необходимости можно скрыть или отобразить эти панели инструментов командой главного меню _View > Toolbars_ или с помощью контекстного меню открытых панелей инструментов (щелчком правой кнопкой мыши).

bq. Примечание: использование палитры *Standart* считается нецелесообразным, поскольку все команды (создание нового документа, сохранение и печать документа, работа с буфером обмена), выполняемые с помощью этой палитры осуществляются с помощью горячих клавиш или главного меню File и Edit.

Панели инструментов можно перемещать по рабочей области программы DreamWeaver, так же можно сделать панели инструментов плавающими. Исключением является панель инструментов Coding, которая всегда располагается вдоль левой границы окна исходного кода документа.

h3. Панель инструментов Document

Панель инструментов Document по умолчанию располагается сверху окна документа и содержит в себе следующие инструменты:

* команды представления документа;
* *Live Code*: отображает реальный код, используемый браузером для отображения страницы;
* *Check Browser Compatibility*: проверка совместимости браузеров;
* *Live View*: отображает документ, подобное представлению документа в браузере с возможностью перехода по ссылкам;
* *Inspect*: режим проверки CSS;
* *Preview/Debug In Browser*: просмотр в браузере;
* *Visual Aids*: вспомогательные элементы;
* *Refresh Design View*: обновление представления «Дизайн» ([F5]);
* поле ввода заголовка документа (html-тег <title>);
* *File Management*: управление файлами.

!05.jpg!

h4. Рис. 5. Панель инструментов Document

Возможно три варианта представления документа:

* *Code*: отображение исходного кода документа;
* *Split*: одновременное отображение исходного кода документа и визуального представления документа в окне документа. По умолчанию код документа расположен слева, визуальное представление – справа (можно расположить области кода и визуального представления горизонтально, сняв «флажок» в главном менюView > Split Vertically, также можно поменять исходный код и визуальное представление местами, указав в главном менюView > Design View on…);
* *Design*: отображение визуального представления документа.

bq. Примечание: для некоторых типы файлов на основе кода, например CSS, JavaScript, XML, невозможно представление *Design*. Такие документы отображаются только в представлении *Code*, при этом кнопки включения режимов *Design* и *Split* становятся неактивны.

Инструмент *File Management* вызывает меню команд управления файлами; большинство этих команд активно только в случае установки локальной и удаленной копий сайта в окне *Site Setup*.

Инструмент *Inspect* помогает визуально понять, где какие проблемы в css. В основном это связано с параметрами width/height, padding и margin. После запуска функции блоки начинают подсвечиваться при наведении на них курсора. Разными цветами подсвечиваются внешний и внутренний отступы, так и высота, и ширина блока. Нажав на блок, мы можем отредактировать свойства css-селектора и сразу же увидеть изменения.

Инструмент *Preview/Debug In Browser* позволяет просмотреть разрабатываемый документ в браузере. Для этого необходимо указать доступные в операционной системе браузеры в окне настроек DreamWeaver. Для быстрого доступа можно задать первичный и вторичный браузеры и вызывать их, используя сочетание клавиш [F12] (Primary Browser) и [Ctrl] + [F12] (Secondary Browser).

Меню инструмента *Visual Aids* управляет отображением вспомогательных визуальных элементов в окне дизайна. При работе в режиме *Code* кнопка неактивна.

Инструмент *Check Browser Compatibility* позволяет проверить документ на совместимость с распространенными типами браузеров. Версии браузеров указываются в настройках Settings меню Check Browser Compatibility.

bq. Примечание: использование этой функции не заменяет  тестирование на совместимость с наиболее распространенными браузерами.

Панель инструментов *Browser Navigation* становится активной при интерактивном просмотре, и в ней отображается адрес страницы, находящейся в окне документа. Панель содержит адресную строку, что позволяет открыть в Dreamweaver любой сайт.

h3. Панель инструментов Coding

Панель инструментов *Coding* содержит кнопки, позволяющие выполнять стандартные операции, облегчающие работу с кодом:

* сворачивание и разворачивание фрагментов кода;
* подсветка неверного кода;
* применение и удаление комментариев;
* выравнивание кода и автоформатирование;
* применение сниппетов (часто используемых фрагментов кода).

Панель *Coding* всегда располагается вертикально в левой верхней части окна представления кода. Ее нельзя переместить или отстыковать, можно только отключить ее видимость.

h3. Палитра Insert

Кнопки панели *Insert* служат для вставки в документ элементов html и элементов серверных включений, а так же элементов фреймфорка SPRY (рассмотрение этих элементов выходит за рамки нашего курса).

Содержимое, отображаемое в инспекторе свойств зависит от выделенного элемента страницы.

При внесении изменений в исходный код страницы содержимое инспектора кода блокируется. Для получения возможности работы с инспектором свойств необходимо нажать кнопку обновления отображения на панели *Document* или в инспекторе свойств, либо нажав клавишу [F5].

Инспектор свойств может отображаться в двух режимах: нормальном и сокращенном. Для переключения между режимами необходимо нажать кнопку в нижнем правом углу инспектора.

h3. Строка состояния

Строка состояния располагается в нижней части окна документа и включает в себя дополнительные средства работы с документом:

* селектор тегов;
* средства управления масштабом отображения;
* средства изменения размера окна документа;
* информацию о размере документа и предположительном времени загрузки страницы в браузере;
* информацию о кодировке текста в текущем документе.

bq. Примечание: средства изменения масштаба отображения и средства изменения размера окна документа доступны только в режимах отображения Design и Split. Селектор тегов и информация о размере документа доступна во всех режимах работы с html-документом. При работе с файлами .css и .javascript доступна только информация о размере документа.

h3. Селектор тегов

Селектор тегов является мощным инструментом для продуктивной работы с иерархией тегов в документе. Инспектор тегов представляет собой цепочку тегов, начиная от тега <head> или <body>, с указанием их классов и идентификаторов. С его помощью можно быстро выделить или удалить теги, присвоить им классы или идентификаторы, изменить тег или его атрибуты.

h3. Инструменты управления масштабом и размером окна документа

Для изменения масштаба отображения документа в окне дизайна можно использовать инструменты *Zoom* и *Hand*, а так же раскрывающийся список значений масштабов.

Для просмотра содержимого страницы при различных разрешениях экрана можно использовать выпадающее меню Sizes. Размеры, содержащиеся в списке можно настроить выбрав пункт Edit Sizes… выпадающего меню. Изменить размер окна документа можно только в случае расположения окон документов каскадом (Window > Cascade). При максимизированном отображении окна документа (используется по умолчанию) пункты выпадающего меню Sizes неактивны.

bq. Примечание: рекомендуется проверять отображение страниц сайта при различных разрешениях, используя тестирование в браузере и изменяя размер окна браузера, поскольку отображение документа в окне дизайна не всегда корректно.

h3. Информация о размере документа

DreamWeaver указывает объем и приблизительное время полной загрузки документа, включая все графические элементы и прикрепленные файлы. Время загрузки документа указывается в соответствии со скоростью соединения, определенной в настройках программы.

h2. Окно документа

По умолчанию в DreamWeaver окна располагаются развернутыми на весь экран, каждому документу соответствует вкладка с названием файла в верхней части окна документа. Используя контекстное меню вкладки (вызывается щелчком правой кнопкой мыши) можно выполнить операции сохранения или закрытия как активного документа, так и всех открытых в DreamWeaver файлов.

Изменить способ расположения окон документов можно щелчком по стандартным кнопкам в правом верхнем углу окна документа, либо используя команды главного меню Window > Cascade, Window > Tile Horizontally и Window > Tile Vertically.

h1. Способы повышения эффективности работы

Если у вас недостаточно высокое разрешение монитора, можно максимизировать рабочую область, скрыв все панели инструментов нажатием горячей клавиши [F4]. Повторное нажатие клавиши [F4] вернет отображение панелей.

Для увеличения отображаемого на экране кода, переключитесь в режим Code и сверните инспектор свойств, нажав сочетание горячих клавиш [Ctrl]+ [F3].

Для быстрого переключения между открытыми документами используйте сочетание клавиш [Ctrl] + [Tab].

Для переключения между режимами *Design* и *Code* используйте сочетание клавиш [Ctrl] + [~]. Это же сочетание клавиш меняет активность окон в режиме *Split* и *Split Code*.

h1. Настройки программы _Adobe DreamWeaver_ CS5

Для изменения настроек программы _Adobe DreamWeaver_ необходимо выполнить команду главного меню _Edit > Preferences…_ ([Ctrl] > [U]).

Окно настроек программы разделено на две области: в левой находится список категорий настроек, в правой — параметры настроек.

* *General*: основные настройки программы;
* Accessibility;
* *АР Elements*: параметры абсолютно позиционированных элементов;
* *Code Coloring*: подсветка кода;
* *Code Format*: форматирование кода;
* *Code Hints*: подсказки в коде;
* *Code Rewriting*: перезапись кода;
* *Copy/Paste*: настройки копирования и вставки
* *CSS Styles*: настройки правил генерации css;
* *Flie Compare*: сравнение файлов;
* *Flie Types / Editors*: соответствие типов файлов, внешние редакторы;
* *Fonts*: шрифты по умолчанию;
* *Highlighting*: подсветка в режиме Design;
* *Invisible Elements*: отображение невидимых элементов;
* *New Document*: параметры нового документа;
* *Preview in Browser*: настройка браузеров;
* *Site*: настройки панели Files;
* *Status Bar*: настройки разрешений экрана и скорости соединения;
* *Validator*: настройки проверки соответствия стандартам.

h2. Раздел Code Format

Default Tag Case и Default Attribute Case: регистр имени тега и атрибутов тега по умолчанию. Обязательно <lovercase> и lovercase='value'. Можно включить параметры «Override Case Of:». В этом случае при открытии документа все теги и названия атрибутов будут приводиться к единому регистру.

В дополнительном окне CSS Format установить параметры:

* Indent properties with 1 tabs;
* Each properties with separate line;
* Blank line between rules.
* Остальные параметры отключить. Такая настройка форматирования позволяет создавать максимально удобочитаемые таблицы стилей.

h2. Раздел Invisible Elements

Некоторые html-элементы, такие как, например, комментарии, не отображаются браузером. Для облегчения работы в режиме дизайна можно установить отображение маркеров для невидимых элементов.

Установим следующие значения:

* *Named Anchors*: именованные якоря;
* *Comments*: комментарии;
* *Embedded Style*: внедренные стили;
* *Hidden Form Fields*: скрытые поля форм;
* *Form Delimiter*: границы форм;
* *CSS display:none*: элементы с отключенной видимостью.

h1. Работа с html-объектами

В Dreamvewer возможно два варианта работы с html-объектами: создание и изменение объектов вручную в окне кода и вставка объектов, используя меню и панель Insert, редактирование при помощи инспектора свойств.

h2. Работа с текстом

По умолчанию все блоки текста, создаваемые в окне *Design*, оформляются как параграф.

Для изменения формата текста можно использовать инспектор свойств, категорию Text в меню Insert, а также горячие клавиши:

* Заголовок 1 уровня: [Ctrl] + [1];
* Заголовок 2 уровня: [Ctrl] +  [2];
* Заголовок 3 уровня: [Ctrl] +  [3];
* Заголовок 4 уровня: [Ctrl] +  [4];
* Заголовок 5 уровня: [Ctrl] +  [5];
* Заголовок 6 уровня: [Ctrl] +  [6];
* Параграф: [Ctrl] + [Shift] + [p];
* Полужирный: [Ctrl] + [b];
* Разрыв строки: [Shift] + [Enter];
* Неразрывный пробел: [Ctrl]+ [Shift] + [Space]

h2. Создание списков

Для создания списков необходимо предварительно отформатировать текст как параграфы. После чего необходимо нажать соответствующую кнопку на панели Insert (категория Text) или в инспекторе свойств.

h2. Создание гиперссылок

Для создания гиперссылки необходимо выделить элемент, который будет являться гиперссылкой и ввести адрес ссылки в поле Link в инспекторе свойств либо нажать кнопку Hyperlink на панели Insert (категория Common).

h2. Работа с изображениями

Для вставки изображения в документ нужно нажать кнопку Image из списка на панели Insert (категория Common) или нажать сочетание клавиш[Ctrl] +[Alt] +[I].

Далее необходимо указать путь до изображения, а так же альтернативный текст и полное описание к изображению (необязательно).

В случае вставки изображения в окне *Design*, в код будут автоматически добавлены значения ширины и высоты. При вставке изображения при активном окне *Code*, ширину и высоту необходимо указать вручную, либо при помощи инспектора свойств.

h2. Работа с таблицами

Для вставки таблицы используйте один из следующих способов:

* Кнопка на панели _Insert_ в разделе _Common_;
* Кнопка на панели _Insert_ в разделе _Layout_;
* Сочетание клавиш [Ctrl] + [Alt] + [T].

В открывшемся диалоговом окне укажите количество строк и столбцов, а так же, при необходимости, расположение заголовочных ячеек.

bq. Примечание: устанавливать какое-либо (даже нулевое) значение для параметров Border, Cellspacing, Celpadding не рекомендуется. Эти величины следует задавать в CSS-описании тега <table>.

h3. Редактирование таблиц

В зависимости от того, какой элемент структуры выделен, будет меняться содержимое инспектора свойств. Наиболее простым способом выделения тегов строки и таблицы является выделение ячейки таблицы в окне дизайна и последующее выделение требуемого тега в селекторе тегов.

Также можно поставить курсор в ячейку в окне Design и нажать сочетание клавиш [Ctrl] +[A], повторное нажатие [Ctrl] +[A] приведет к выделению всей таблицы.

Для упрощения операций над таблицами можно переключиться в режим Expanded View в категории Layout панели Insert.

h1. Работа с CSS

Возможно несколько способов работы с каскадными таблицами стилей в _Adobe DreamWeaver_:

* создание и правка кода вручную;
* использование палитры CSS Styles;
* использование диалогового окна CSS Rule Definition.

Каждый подход имеет свои достоинства и недостатки. Обычно комбинируют все три метода работы.

h2. Палитра CSS Styles

Палитра CSS Styles служит для отображения и изменения информации о имеющихся стилевых описаниях.

Палитра CSS Styles может находиться в одном из двух режимов:

* список всех использующихся стилевых описаний;
* список стилевых описаний для выделенного элемента.

h3. Режим All

В режиме отображения всех примененных к документу стилей, палитра CSS Styles делится на две части. В верхней располагается «дерево» селекторов, описанных во внешних и внутренних таблицах стилей, а так же во внедренных стилевых описаниях.

В нижней части отображается список свойств и значений для выбранного селектора. При щелчке по значению свойства становится активным список выбора значений свойства. Так же можно добавить новые свойства, щелкнув по пункту Add Property.

Используя контекстное меню можно копировать, вставлять и удалять селекторы, свойства и значения.

h3. Режим Current

В режиме Current в палитре CSS Style отображается информация обо всех имеющихся свойствах для выбранного html-элемента. Палитра CSS Styles в этом режиме делится на три части.

В верхней части отображается список пар свойство-значение для выделенного элемента.

!24.jpg!

h4. Рис. 24. Палитра CSS Styles

Средняя область палитры может отображать справочную информацию о выделенном свойстве, либо показывать каскад зависимостей для выделенного элемента.

bq. Примечание: В случае работы с файлами .css верхняя и средняя области палитры становятся пустыми, так как отсутствует понятие выделенного html-элемента.

В нижней части отображается список свойств и значений для выбранного в средней части селектора. При щелчке по значению свойства становится активным список выбора значений свойства. Так же можно добавить новые свойства, щелкнув по пункту Add Property. В случае, если свойства, описанные для выбранного селектора, перекрываются другими стилевыми описаниями, название свойства отображается зачеркнутым.

Действия, осуществляемые с помощью палитры CSS Style

При помощи кнопок, расположенных в нижней части палитры CSS Styles можно переключать вид отображения списка свойств и значений селекторов:

* *по категориям*. Свойства группируются по категориям, для используемых свойств указывается значение.
* *по алфавиту*. Все свойства отображаются в алфавитном порядке, для используемых свойств указывается значение.
* *только установленные свойства*. Отображаются только применяемые к элементу свойства и их значения.

Так же при помощи палитры CSS Styles возможно прикреплять таблицы стилей к документу, создавать, редактировать и удалять правила CSS.

h2. Прикрепление новой таблицы стилей

Для присоединения к документу таблицы стилей необходимо указать в диалоговом окне ее путь, тип присоединения и, при необходимости, тип устройства отображения.

Так же возможен предварительный просмотр результатов присоединения таблицы стилей.

h2. Создание нового правила CSS

Для создания нового правила CSS необходимо указать в диалоговом окне тип создаваемого селектора, а так же место определения селектора: внешняя, либо внутренняя таблица стилей.

Настройка параметров стиля, а так же их правка, производится в окне CSS Rule Definition.

h2. Диалоговое окно CSS Rule Definition

Настройка параметров стилевого описания можно осуществлять в диалоговом окне CSS Rule Definition.

Диалоговое окно разделено на две части, в левой расположенный категории CSS-свойств, справа — поля выбора свойств и их значений.

Все CSS-свойства разделены на 8 категории:

* *Type*: параметры шрифта;
* *Background*: параметры фона;
* *Block*: свойства абзаца
* *Box*: парамтры блочной модели ;
* *Border*: параметры рамки;
* *List*: параметры списка;
* *Positioning*: позиционирование;
* *Extensions*: дополнительные параметры. Разрыв печатной страницы, изменение свойств курсора и фильтры.

h1. Создание сайта

Для того, что бы DreamWeaver воспринимал сайт как единое целое, а не набор разрозненных документов, необходимо создать DreamWeaver Site. После создания сайта станет возможно:

* устанавливать адресацию документов относительно корня сайта;
* использовать шаблоны;
* осуществлять глобальный поиск и замену;
* производить синхронизацию локальной и удаленной копий сайта;
* проверять целостность ссылок;
* использовать общие ресурсы сайта (Assets).

h2. Определение сайта

Для определения сайта в DreamWeaver необходимо:

* выполнить команду главного меню _Site > New Site…_;
* в категории Site и указать имя сайта и путь к корневой папке сайта;
* при необходимости перейти в категорию Advanced и настроить прочие параметры.

h3. Категория Cloaking

*Cloaking* — сокрытие файлов от проведения автоматических операций, производимых DreamWeaver над файлами сайта. К таким операциям относятся поиск и замена, публикация и синхронизация файлов.

В категории Cloacing окна Site Definition можно определить типы файлов, которые будут скрыты от DreamWeaver. Например, «.fla .psd .ai .doc .pdf .xls».

h3. Раздел Design Notes

DreamWeaver позволяет сохранять с файлом произвольные заметки разработчика. Заметки хранятся в папе _notes. Если вы не используете заметки разработчика, можно отключить опцию Maintain Design Notes.

h2. Палитра Files

Палитра Files позволяет производить любые операции над файлами в локальной файловой системе компьютера. В случае если вами определен DreamWeaver Site, вы можете дополнительно производить операции загрузки и выгрузки файлов на сервер, а так же осуществлять синхронизацию файлов локальной и удаленной копий сайта.

h3. Синхронизация файлов

При проведении операции синхронизации файлов DreamWeaver можно указать для синхронизации весь сайт или только выбранные папки и файлы, а также дополнительные параметры синхронизации:

* разместить новые файлы на удаленном сервере;
* получить все новые файлы с удаленного сервера;
* получить и разместить все новые файлы.

В последнем случае на сервер будут отправлены более новые файлы из локальной копии, а с сервера скачаны более новые файлы из удаленной копии.

h2. Поиск и замена

DreamWeaver обладает мощной системой поиска и замены текста, исходного кода, а так же отдельных тегов и атрибутов.

Поиск и замену можно проводить:

* в выделенном тексте;
* в открытом файле;
* во всех открытых файлах;
* в отдельной папке на диске;
* над выделенными файлами на текущим сайте;
* над всеми файлами на сайта.

bq. Примечание: возможно осуществление поиска и замены текста над неоткрытыми в DreamWeaver файлами. Но стоит помнить о том, что такие операции не могут быть отменены в дальнейшем.

Для поиска фрагмента текста необходимо нажать горячую клавишу[Ctrl] +[F], указать область поиска, указать тип поиска (исходный код, текст, определенный тег), ввести в поле Find искомый текст и нажать Find. Найденный фрагмент будет выделен в текущем окне представления документа. Если необходимо найти все совпадение — нажмите кнопку Find All. Результаты поиска будут отображены в палитре Result. Двойным щелчком по результату можно перейти к найденному фрагменту.

!29.jpg!

h4. Рис. 29. Диалоговое окно Find and Replace

Для замены фрагмента текста или определенного тега — укажите требуемые параметры в поле Replace.

Дополнительно можно указать критерии поиска:

* *Match case*: учитывать регистр;
* *Match whole word*: только слово целиком;
* *Ignore whitespaces*: игнорировать пробелы;
* *Use regular expression*: использовать регулярные выражения (маски текста).

Так же можно сохранять параметры поискового запроса и загружать сохраненные.

h1. Тестирование и проверка

Тестирование сайта и его проверка включает в себя:

* Проверку на соответствие стандартам (_Validation_);
* Проверка на совместимость браузеров (_Browser compatibily check_);
* Проверка целостности ссылок (_Link check_).

h2. Проверка на соответствие стандартам

Для проверки на соответствие стандартам необходимо указать стандарт, в соответствии с которым осуществлялась разработка страницы. Для html-документов стандарт указывается в теге <DOCTYPE>. В случае, когда версия стандарта не указана, тестирование проводится в соответствии с параметрами, указанными в настройках DreamWeaver.

Проверка на соответствие стандартам осуществляется выбором соответствующего пункта в меню панели Validation (Validate Current Document, Validate Entire Current Local Site, Validate Selected Files in Site). В результате проверки на соответствие стандартам в палитре Validation будут выведены результаты.

Результаты включают в себя информацию об ошибках, предупреждения, а так же сообщения об ошибках вложенности.

Сделав двойной щелчок по ошибке, вы перейдете к строке кода содержащей ошибку.

Наиболее часто встречающиеся ошибки:

* опечатки;
* незакрытые теги;
* неправильная вложенность тегов;
* неправильное употребление специальных символов.

Повторную валидацию желательно проводить после каждой исправленной ошибки, потому что многие ошибки являются следствиями предыдущих ошибок.

bq. Примечание: создание страниц, максимально совместимых с разными браузерами, и одновременно совместимых со стандартами, не всегда возможно.

h2. Проверка ссылочной целостности

Проверка ссылочной целостности позволяет определить:

* *Broken Links* («Битые» ссылки): ссылки на несуществующие документы;
* *External Links* (Внешние ссылки): ссылки на другие сайты и адреса электронной почты. Эти ссылки могут оказать неверными в любой момент;
* *Orphaned Files* (Страницы-сироты): страницы, на которые нет ни одной ссылки.

Для проверки ссылчной целостности необходимо выбрать требуемый пункт в меню палитры *Link Checker* и указать параметры отображения результатов проверки.
