Требования к домашним работам и курсовой работе

Все файлы, создаваемые в процессе выполнения самостоятельных работ, домашние задания и итоговая работа должны соответствовать указанным ниже требованиям. Невыполнение указанных требований может служить основанием для непринятия работы преподавателем.

Общие замечания

Если не сказано иначе, то HTML документ должен соответствовать XHTML 1.0 Transitional DTD.

Кодировка текстовых файлов - только UTF-8

Не следует использовать p для описания не абзаца, а просто элемента уровня блока. Используйте для этого div

Не следует использовать заголовок n-го уровня, если отсутствует предшествующий ему заголовок (n-1)-го уровня. Для выделения текста используйте стили

Не следует использовать таблицы там, где без них можно обойтись. Старайтесь реализовать задачу средствами CSS

Именование и расположение файлов

Имя файла должно содержать только латинские буквы, цифры, символы «_» и «-». Присутствие кириллицы и символов пробела недопустимо
Имя и расширение файла должно быть записано в нижнем регистре

Расширение файла: 

 html-докумены: .html или .htm

 Таблицы стилей: .css

 Графические изображения: .jpg, .png, .gif

Графические изображения должны находиться в каталоге «img»

Файлы таблиц стилей должны находиться в каталоге «css»

Форматирование документов

Файлы html и css должны проходить валидацию

Специфичные свойства, задаваемые специально для Internet Explorer должны быть вынесены в отдельный файл, подключаемый через 
условные комментарии

Основные блоки разметки и файлов CSS должны быть отмечены комментариями

Не рекомендуется описывать стили и скрипты в теле документа, используйте для этого внешние файлы

Форматирование html-файлов

Все тэги и их атрибуты должны быть записаны строчными буквами

Все значения атрибутов записываются в кавычках (двойных или одинарных). Рекомендуется использовать единый стиль (т.е. или двойные, или одинарные)

Все элементы должны иметь закрывающие тэги. Элементы, не могущие иметь содержимое, следует записывать в следующем виде:

<elem [attr1='value' attr2='value'...]/>

Пары атрибут/значение обязаны быть записаны полностью, т.е. атрибуты подобные selected, должны быть записаны в виде selected='selected'

Вложенность элементов должна быть показана при помощи отступов

Форматирование css-файлов

Каждый селектор должне записываться на отдельной строке, без отступа от левого края

Каждая пара «свойство: значение;» должна записываться на отдельной строке

Пары «свойство: значение;» должны записываться с отступом от левого края

Элемент head

В шапке следует использовать указанный порядок следования элементов:

title с кратким и достаточным описанием документа. Рекомендуется использовать вид подобный следующему: Страница - [Подраздел] - [Раздел] - Сайт
title должен отражать содержимое документа. Использование <title>Untitled Document</title> запрещено
